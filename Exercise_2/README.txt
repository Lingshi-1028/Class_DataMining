===电影评价数据协同过滤推荐项目===

一、项目概述
该项目基于协同过滤思想，采用 SVD 矩阵分解算法实现电影评分数据的推荐模型构建，并通过 10 折交叉验证量化评估模型性能（RMSE、MAE 指标），流程简洁高效，结果可直接落地参考。

核心功能
1、加载指定目录下的电影评分 CSV 数据；
2、初始化 SVD 协同过滤算法；
3、10 折交叉验证评估模型，输出 RMSE、MAE 指标；
4、计算并打印验证结果的平均值，直观反映模型预测精度。

二、环境依赖
必备库
pandas：加载与处理 CSV 评分数据；
surprise：实现协同过滤算法与交叉验证（核心推荐库）。

安装命令
pip install pandas scikit-surprise
注：surprise库全称scikit-surprise，部分环境需直接安装该全称包，兼容 Python 3.7 及以上版本。

三、数据与文件准备
1、项目文件结构：

Exercise_2/
├── data/
│   └── ratings.csv  # 核心评分数据文件
└── 电影协同过滤.py  # 本项目代码文件

2、数据要求（ratings.csv）：
	必须包含三列：userId（用户 ID）、movieId（电影 ID）、rating（评分）；
	评分范围适配(0.5, 5.0)，符合常规电影评分场景；
	格式为 CSV，表头对应上述三列名称，无额外无关列（避免数据加载报错）。

四、核心模块说明
1、路径配置：通过os.path.join拼接根目录与数据目录，指定ratings.csv路径，便于目录迁移；
2、数据加载：先用pandas读取 CSV，再通过surprise的Dataset.load_from_df转换为算法兼容格式，Reader定义评分范围；
3、算法初始化：选用SVD算法（稳定高效，适合评分预测类协同过滤场景）；
4、交叉验证：cross_validate实现 10 折验证，评估RMSE（均方根误差）、MAE（平均绝对误差）；
5、结果输出：计算并打印验证结果的平均 RMSE 与 MAE，保留 4 位小数，便于对比分析。

五、运行步骤
1、按上述文件结构整理目录，将ratings.csv放入data文件夹；
2、修改root_dir为你的实际Exercise_2目录路径（确保路径格式正确，Windows 系统注意转义符）；
3、运行代码：
	命令行：python 电影协同过滤.py（替换为你的代码文件名）；
	IDE：直接点击运行按钮；
4、查看结果：控制台先输出每折验证的详细指标，最后输出平均 RMSE 与 MAE。

六、结果说明
1、指标解读（值越小越好）：
	RMSE：对预测误差较大的值更敏感，反映模型整体预测偏差；
	MAE：反映预测值与真实值的平均绝对偏差，更稳健；
2、输出格式：

10折交叉验证最终结果：
平均RMSE: x.xxxx
平均MAE: x.xxxx

七、注意事项
1、路径配置：Windows 系统目录路径使用r"路径"避免转义错误，Linux/Mac 系统直接使用常规斜杠路径；
2、库安装问题：若pip install scikit-surprise失败，可先安装依赖pip install numpy scipy，再重新安装；
3、数据格式：确保ratings.csv的三列表头名称完全匹配userId、movieId、rating，无大小写错误；
4、算法替换：若需更换协同过滤算法，可将SVD()替换为KNNBasic()（近邻协同过滤），无需修改其他流程。
